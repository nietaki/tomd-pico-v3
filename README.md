## How does it work?

The whole system is first prototyped in an Elixir livebook using a
Postgres db as storage.

I came up with the questions on my own, aiming to have them applicable
to many headwords, easy to answer by most people, and not too repetitive.

The answers to `question x headword` combinations have been obtained by prompting a local Ollama instance running a small model (currently `qwen3-14b` and `mistral-small3.2-24b`) - the whole [`game.data.cpp`](/game_data.cpp) is
generated by the livebook
